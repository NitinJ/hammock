Translate the design spec into a concrete implementation specification.

The impl spec should:
- Reference the bug report and design spec, then enumerate the specific files, classes, and functions to be added, modified, or removed.
- Specify the surface-level changes (signatures, fields, return types, branches added) without writing actual code.
- Identify which tests must be added or updated and what they assert.
- Note any migration, configuration, or rollout considerations.

Verify each file path and symbol you name actually exists in the working directory. If the design spec referenced something that turns out not to exist, raise the discrepancy instead of papering over it.
